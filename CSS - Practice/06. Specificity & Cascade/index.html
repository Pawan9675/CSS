📌 Understanding Specificity

🔹 Specificity values:

  • 🌍 Universal Selector (*): 0
  • 🔹 Element Selector (e.g., h1, p): 1
  • 🔹 Class Selector (e.g., .class): 10
  • 🔹 Attribute Selector (e.g., [data-attribute]): 10
  • 🔹 Pseudo-class (e.g., :hover): 10
  • 🔹 ID Selector (e.g., #id): 100
  • 🔹 Inline Style: 1000
  • ⚠️ !important: Overrides all specificity values

🧮 Specificity Calculation Example

Selector: a.link.class1.class2[href]:hover

📊 Calculation:

  • 🔹 Element Selector (a): 1
  • 🔹 Class Selectors (.link, .class1, .class2): 10 + 10 + 10 = 30
  • 🔹 Attribute Selector ([href]): 10
  • 🔹 Pseudo-class (:hover): 10

📌 Total Specificity: 1 + 30 + 10 + 10 = 51

⚡ CSS Cascade Algorithm

📝 CSS stands for Cascading Stylesheets. The cascade determines which rule applies when multiple CSS rules affect the same element.

🛠️ The cascade algorithm follows **4 stages**:

  • 1️⃣ Position & Order: The order in which CSS rules appear.
  • 2️⃣ Specificity: Determines the strongest selector match.
  • 3️⃣ Origin: Defines where CSS comes from (browser styles, extensions, or authored styles).
  • 4️⃣ Importance: Some rules (e.g., !important) have higher weight.

🔍 Specificity Calculation Formula

💡 To calculate specificity, assign values:

Universal Selector:        0
Element & Pseudo-elements: 1
Class, Attribute, Pseudo-class: 10
ID Selector:              100
Inline Style:             1000

📢 Priority Order:

1️⃣ Inline Style > 
2️⃣ ID Selector > 
3️⃣ Class / Attribute / Pseudo-class > 
4️⃣ Element Selector > 
5️⃣ Universal Selector

📌 Summary

✅ The final color applied depends on the highest specificity.

✅ If specificity is equal, the last rule in CSS is applied.
