CSS Sizing Units Explained

In this example, we will explore the different absolute and relative units of measurement in CSS, including their behavior and use cases.

Absolute Units:

Absolute units provide fixed sizes, regardless of the viewport or parent element's size. They are typically used for precise control over an element's dimensions. Here are the most commonly used absolute units:

  • cm - Centimeters (1 cm = 37.8 px)
  • mm - Millimeters
  • in - Inches (1 in = 96 px = 2.54 cm)
  • pt - Points (1 pt = 1/72 of 1 in)
  • pc - Picas (1 pc = 1/6 of 1 in)
  • px - Pixels (1px = 1/96 of 1 in)

Relative Units:

Relative units depend on the size of other properties, such as the parent element's font size or the viewport dimensions. These are useful for creating flexible and responsive designs:

  • em - Relative to the font size of the parent element
  • rem - Relative to the font size of the root element
  • vw - Relative to 1% of the viewport width
  • vh - Relative to 1% of the viewport height
  • % - Relative to the parent element's size
  • vmin - Relative to 1% of the smaller dimension of the viewport (either width or height)
  • vmax - Relative to 1% of the larger dimension of the viewport (either width or height)
  • ch - Relative to the width of the character "0"
  • ex - Relative to the height of the letter "x" in the current font

Other Metrics:

We can also use other metrics such as Opacity, RGB & RGBA, and HSL & HSLA for styling and color manipulations:

  • Opacity: Ranges from 0 (transparent) to 1 (opaque)
  • RGB & RGBA: Ranges from 0 to 255 for Red, Green, and Blue. RGBA allows you to control opacity (alpha channel) from 0 (transparent) to 1 (opaque).
  • HSL & HSLA: HSL uses Hue (0-360), Saturation (0-100%), and Lightness (0-100%). HSLA also includes alpha (opacity) values.
